    • Ghost

    A 2D Platformer Game developed with Phaser 3

    Project Background

    Ghost is a 2D platformer game created as part of a university project in Medieninformatik. It was designed to explore gameplay mechanics like movement, jumping, traps, and shooting, using the Phaser 3 framework. The project also aimed to strengthen skills in level design, animation integration, and game scene management.

    Context

    This project was developed during a course module focused on interactive media and game development. The primary goal was to implement a complete and functional browser-based game, including multiple levels, sound, animations, and basic player interaction.

    Process

    • Planning: Defined game scope, feature list, and initial level sketches.
    • Asset sourcing: Used royalty-free pixel art and sound effects.
    • Implementation: Developed the game incrementally:
      • Core player movement (jumping, walking, gravity)
      • Level tiles and collisions
      • Traps, portals, enemies
      • Shooting mechanic and soul pickups
      • Scene transitions and UI feedback
    • Testing: Regularly tested game behavior and fixed bugs related to physics, overlapping scenes, or player animation glitches.

    Research

    A questionnaire was distributed to potential users to gather insights on their wardrobe habits and expectations for such an app. Key findings included:

    • Technical research: Phaser 3 documentation, tutorials, and community forums were explored to understand best practices in scene management, physics, collisions, and animations.
    • UX/Game design: I researched classic and indie platformers (like Celeste, Hollow Knight) to analyze level pacing, challenge design, and player feedback systems.
    • Audio integration: Studied how to trigger sound effects based on player input and scene events.

    These insights guided design priorities and feature selection.

    Design Goals

    • Create a fun and engaging platforming experience.
    • Keep the visual style minimal and pixel-art-based.
    • Implement smooth player controls and animations.
    • Build levels with increasing difficulty and diversity.
    • Provide audio feedback for player actions and scene changes.

    Design Problems

    • Creating reusable and scalable scene architecture in Phaser.
    • Managing tilemap layers and invisible object properties in Tiled.
    • Ensuring collisions and physics interactions worked consistently across different objects.
    • Keeping the game visually readable despite a dark pixel-art theme.

    Design Decisions

    • Split scenes into folders (core, ui, levels) to improve code organization.
    • Used Phaser Groups for spikes, portals, souls, and enemies to simplify object spawning and collision handling.
    • Added a custom game state (GameState.soulAmmo) to handle logic across scenes.
    • Controlled game pacing using timers and tweens (e.g. enemy speed increases over time).
    • Kept UI minimal with pixel fonts to match the visual theme.

    Reflections

    This project helped me understand the technical and design challenges behind creating a complete 2D platformer. I gained experience working with Phaser 3, managing multiple game scenes, handling collision logic, and structuring game code in a modular way. I also improved my debugging and problem-solving skills, especially when dealing with scene transitions or game state errors. "Ghost" gave me a foundation I plan to build on for future game prototypes or interactive media projects
